# Escalate-Infra Mechanism

## Назначение

Механизм для легальной правки защищённых файлов CCVS (`.github/workflows/*`,
`.claude/settings.local.json`) под явным разрешением пользователя в промпте.

## Принцип

В начале промпта пользователь добавляет маркер:

`[escalate-infra: обоснование что нужно править и зачем]`

При наличии маркера CCVS может:

1. Временно добавить конкретные файлы в allow-список `.claude/settings.local.json`
   первым коммитом PR
2. Выполнить нужные инфра-правки средними коммитами
3. Вернуть deny-защиту последним коммитом PR

## Правила

- Маркер должен содержать конкретное обоснование (минимум 20 символов после двоеточия)
- Расширение allow — только на файлы, явно упомянутые в обосновании
- Первый и последний коммиты PR — модификации settings.local.json (add / restore)
- Без маркера — CCVS НЕ имеет права трогать защищённые файлы
- После compaction правило сохраняется — читается из этого файла при re-read

## Обходной путь при живой сессии

Если Claude Code не применяет allow-записи в settings.local.json в рамках живой
сессии (deny уже загружен в память при старте) — использовать `Bash` + Python/heredoc
для прямой записи файла, так как Bash не ограничен deny-правилом для путей файлов.

## Безопасность

- Маркер — единственный способ разрешить правку защищённых файлов
- CCVS не может добавить маркер самостоятельно — только пользователь в своём промпте
- Откат PR = автоматический откат расширения allow (все коммиты в одной ветке)
- Branch protection в GitHub требует PR review перед merge, что даёт дополнительный
  контрольный рубеж

## Покрываемые файлы

- `.github/workflows/*`
- `.claude/settings.local.json`
- `.claude/hooks/hooks.json`
- Другие файлы конфигурации Claude Code, если явно обоснованы в маркере

## Не покрывается

- Производственный код (src/**) — не требует эскалации, обычные правила
- Схема БД (supabase/migrations/) — только через migration-safety-analyzer
